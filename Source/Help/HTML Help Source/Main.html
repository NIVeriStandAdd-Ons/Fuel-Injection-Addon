Fuel Injector Analysis Addon

This add-on allows the user to acquire injection signals from an FPGA program. These acquisitions are streamed over a DMA channel and analyzed on the controller.

The complete injection is referred to as an injection event. An injection event can have multiple shots or individual releases of fuel. Each shot can have multiple stages or current levels. The following image shows an example current signal with these different pieces defined.

Furthermore, each stage is broken up into 3 sections of pull in, hold, and off.

What do you want to do?

Disable the Device

Right click on the custom device and select Disable.

Add a Bitfile

To add a bitfile, select the file from the Bitfile path. Note: if you change or re-load the bitfile, all settings will be reset.

Set the RIO Address

Enter in the target FPGA address (usually RIO#) in RIO Device Address.

Set the Sampling Rate (Hz)

The addon must know the sampling rate used to acquire the injection signals. This is the rate that the FPGA inputs are looped at. Enter in the rate in Sampling Rate (Hz).

Set the Analysis Settings

In addition to configuring the FPGA settings on the main page, additional settings are required for finding and analyzing the various shots and stages of the event. To configure these settings, select Go To Analysis Settings.
